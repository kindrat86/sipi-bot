Every integration uses the same core call — evaluate(amount, merchant, category) returning APPROVED, BLOCKED, or FLAGGED — over a zero-dependency client. Open-source core (MIT), free to self-host, or $99/mo hosted with a live dashboard and tamper-evident audit log.

How every integration works under the hood

No matter which framework you use, the wire protocol is identical: your agent sends one POST /v1/transactions/evaluate request with the amount, merchant, and category of the spend it is about to make, and sipi.bot answers with a decision object — APPROVED, BLOCKED, or FLAGGED — plus a human-readable reason.

The Python client (sipi_guard.py) and the TypeScript client (sipiGuard.ts) are thin wrappers around that single call, so there is no SDK lock-in and nothing to keep in sync when a framework updates.

Behind the endpoint, the firewall evaluates six rule families — per-transaction caps, daily totals, velocity limits, merchant allow/block lists, category limits, and time windows — plus an approval threshold for human review. The decision engine is deterministic (no ML inference) and its behaviour is verified by a public eval suite of 53 labeled scenarios, all 53 passing.

Not using one of these frameworks?

You still have three native surfaces. Any runtime that can make an HTTP call can use the API directly. Agents running inside Claude Code, Cursor, or Hermes can call sipi.bot as a native MCP tool. And the CLI covers scripts and cron-driven automations. If you want to kick the tires first, the live playground evaluates real transactions against the production endpoint with no API key, and the risk calculator estimates your worst-case agent spend before you wire anything up.

Frequently asked

Which agent frameworks does sipi.bot support?

sipi.bot works with LangChain, LangGraph, CrewAI, the OpenAI Agents SDK, and the Vercel AI SDK today, plus any runtime that can make an HTTP call or use an MCP tool.

Is there one client for all frameworks?

Yes. Python frameworks use the zero-dependency sipi_guard.py; TypeScript frameworks use sipiGuard.ts. Both wrap the same /v1/transactions/evaluate endpoint.
